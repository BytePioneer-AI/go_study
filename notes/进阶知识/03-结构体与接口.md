# 05 - 结构体与接口

## 本节目标

这一节先掌握两个最核心的问题：

1. Go 的 `struct` 到底是什么，和“类”有什么不同
2. Go 的接口为什么叫“隐式实现”，以及这对写代码有什么影响

如果你之前写过 Java、C#、TypeScript，这一节最重要的心智转换是：

**Go 有结构体和接口，但没有“类 + 继承”那套中心模型。**

Go 更偏向：
- 用 `struct` 组织数据
- 用方法表达行为
- 用 `interface` 描述“需要什么能力”
- 用组合代替继承

---

## 1. 结构体：Go 组织数据的主要方式

在很多面向对象语言里，你可能习惯用 class 同时表示：
- 数据
- 方法
- 继承关系

Go 会把这些拆开。

先看最基本的结构体：

```go
package main

import "fmt"

type User struct {
	Name string
	Age  int
}

func main() {
	user := User{Name: "Tom", Age: 20}
	fmt.Println(user.Name)
	fmt.Println(user.Age)
}
```

这里的 `User` 只是一个数据类型。

你可以先把它理解成：
- 类似其他语言里的“只有字段的数据对象”
- 但它本身不是 class
- 也没有构造器、继承、访问修饰符那一整套机制

### 和其他语言的对比

#### Java 里你可能这样想

```java
class User {
    String name;
    int age;
}
```

#### Go 里更像是

```go
type User struct {
	Name string
	Age  int
}
```

但 Go 不强调“把一切都塞进类里”，而是把：
- 数据定义在 `struct`
- 行为用方法单独挂上去

这和上一节的方法接收者正好连起来。

---

## 2. 结构体可以有方法，但结构体不是类

看例子：

```go
package main

import "fmt"

type User struct {
	Name string
}

func (u User) Greet() string {
	return "Hello, " + u.Name
}

func main() {
	user := User{Name: "Gopher"}
	fmt.Println(user.Greet())
}
```

这里 `Greet()` 是挂在 `User` 上的方法。

所以你会看到一种“像类”的感觉，但要注意本质区别：

- Go 方法是通过 receiver 挂到类型上
- 不是写在 class 内部
- Go 没有继承树来组织类型关系

这意味着：

**Go 的核心不是“对象是什么类的”，而是“这个值有哪些方法、能不能满足某个接口”。**

这个思路马上就会进入接口部分。

---

## 3. 嵌套结构体：Go 更常用组合，而不是继承

很多语言里，复用常见做法是继承：
- `Admin extends User`
- `Dog extends Animal`

Go 不提供 class 继承。

Go 更常见的是 **组合（composition）**。

```go
package main

import "fmt"

type Profile struct {
	City string
}

type User struct {
	Name    string
	Profile Profile
}

func main() {
	user := User{
		Name:    "Tom",
		Profile: Profile{City: "Shanghai"},
	}

	fmt.Println(user.Name)
	fmt.Println(user.Profile.City)
}
```

这里不是“User 继承 Profile”，而是：

**User 拥有一个 Profile 字段。**

这就是组合思路：
- 需要什么能力/数据，就把它作为字段组合进来
- 不靠继承层级复用

### 为什么 Go 偏爱组合

因为继承容易带来：
- 类型关系过深
- 父类耦合过强
- 子类行为边界不清晰

Go 更希望关系简单、显式。

你可以先记一句话：

**Go 里优先考虑“has-a（有一个）”，而不是 “is-a（是一个父类的子类）”。**

---

## 4. 接口：描述“能力”，不是“血统”

这是 Go 非常有代表性的设计。

先看一个接口：

```go
package main

import "fmt"

type Speaker interface {
	Speak() string
}

type Dog struct {
	Name string
}

func (d Dog) Speak() string {
	return "wang: " + d.Name
}

func main() {
	var s Speaker
	s = Dog{Name: "Buddy"}
	fmt.Println(s.Speak())
}
```

这里：
- `Speaker` 是一个接口
- 它要求实现 `Speak() string`
- `Dog` 只要有这个方法，就可以被当成 `Speaker`

### 接口的本质

接口不是在描述“它是什么类型”，而是在描述：

**它能做什么。**

这和很多面向对象语言的思路不完全一样。

---

## 5. Go 接口最关键的特点：隐式实现

这是你必须掌握的重点。

在 Java / C# 里，通常要显式写：

```java
class Dog implements Speaker {
    ...
}
```

Go 不需要。

Go 只要方法对上，就自动算实现了接口。

也就是说，下面这段代码里：

```go
type Speaker interface {
	Speak() string
}

type Dog struct{}

func (d Dog) Speak() string {
	return "wang"
}
```

虽然你**没有写**“Dog implements Speaker”，但 `Dog` 已经实现了 `Speaker`。

### 这带来的好处

1. **解耦更强**：类型不需要提前声明“我要实现哪个接口”
2. **接口更灵活**：谁刚好有这个方法，谁就能用
3. **更适合小接口**：一个接口只表达一个很小的能力

Go 的这个设计非常强调：

**面向接口编程，但接口应该由使用方定义，而不是实现方绑定死。**

---

## 6. 接口值里装的是什么

这一点很多初学者容易模糊。

```go
var s Speaker
s = Dog{Name: "Buddy"}
```

这里 `s` 的静态类型是 `Speaker`，但它里面装的是一个具体值 `Dog{Name: "Buddy"}`。

所以：
- 接口让你用统一方式调用 `Speak()`
- 但运行时真正干活的，还是具体类型 `Dog`

你可以先把接口理解成：

**一个统一的“调用门面”。**

调用：

```go
s.Speak()
```

实际上会落到具体类型的方法上。

---

## 7. 一个更实用的例子：接口让函数不依赖具体类型

```go
package main

import "fmt"

type Speaker interface {
	Speak() string
}

type Dog struct{}

func (d Dog) Speak() string {
	return "wang"
}

type Cat struct{}

func (c Cat) Speak() string {
	return "miao"
}

func printSpeech(s Speaker) {
	fmt.Println(s.Speak())
}

func main() {
	printSpeech(Dog{})
	printSpeech(Cat{})
}
```

这里最值得你体会的是：

`printSpeech` 不关心传进来的是 `Dog` 还是 `Cat`，它只关心一件事：

**你有没有 `Speak() string` 这个能力。**

这就是接口的价值：
- 降低函数和具体类型的耦合
- 让代码更容易扩展

如果以后新增一个 `Person`，只要也实现 `Speak() string`，`printSpeech` 完全不用改。

---

## 8. Go 为什么喜欢小接口

你后面会在标准库里经常看到很小的接口，例如：
- `io.Reader`
- `io.Writer`

它们通常只要求 1 个或少数几个方法。

这是 Go 的一个重要习惯：

**接口越小，复用性通常越强。**

比如一个接口如果只要求：

```go
type Reader interface {
	Read(p []byte) (n int, err error)
}
```

那很多不同东西都可以实现它：
- 文件
- 网络连接
- 字符串读取器
- 内存缓冲区

如果接口设计得很大，反而很难被复用。

所以你刚开始学习时，先建立这个感觉：

- 不要动不动就定义“大而全”的接口
- 优先定义“当前函数真正需要的最小能力”

---

## 9. 结构体 + 接口 + 组合，是 Go 取代传统继承体系的关键组合

这一节你可以先把 Go 的类型系统粗略理解成这样：

1. 用 `struct` 放数据
2. 用方法给类型增加行为
3. 用 `interface` 抽象能力
4. 用组合复用数据和功能

也就是说，Go 没有走：

`类 -> 继承 -> 抽象基类`

而更偏向：

`结构体 -> 方法 -> 小接口 -> 组合`

这就是 Go 风格最重要的差异之一。

## 练习题

目标：

练习结构体、方法、接口，以及隐式实现接口。

要求：

1. 定义接口 `Describer`
2. 定义 `Book` 和 `Movie` 两个结构体
3. 分别实现 `Describe() string`
4. 写函数 `printDescription(d Describer)`
5. 在 `main()` 中分别传入 `Book` 和 `Movie`

### 参考答案代码

```go
package main

import "fmt"

type Describer interface {
	Describe() string
}

type Book struct {
	Title string
}

type Movie struct {
	Name string
}

func (b Book) Describe() string {
	return "Book: " + b.Title
}

func (m Movie) Describe() string {
	return "Movie: " + m.Name
}

func printDescription(d Describer) {
	fmt.Println(d.Describe())
}

func main() {
	book := Book{Title: "Go 入门"}
	movie := Movie{Name: "Interstellar"}

	printDescription(book)
	printDescription(movie)
}
```

### 预期运行结果

```text
Book: Go 入门
Movie: Interstellar
```

## 小结

这一节先记住四句话：

1. **`struct` 是 Go 组织数据的核心方式，但它不是 class**
2. **Go 接口描述的是能力，不是继承关系**
3. **接口实现是隐式的：方法对上就算实现**
4. **Go 更偏向组合，而不是继承**

如果你把这四句话吃透，后面看标准库会顺很多。
