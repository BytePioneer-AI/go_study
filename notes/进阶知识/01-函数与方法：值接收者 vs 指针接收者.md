# 01 - 函数与方法：值接收者 vs 指针接收者

## 本节目标

这一小节先掌握两个最核心的问题：

1. Go 的方法是怎么挂到类型上的
2. 什么时候用值接收者，什么时候用指针接收者

这部分很重要，因为它会影响：
- 方法能不能修改原对象
- 调用时是否发生拷贝
- 后面接口是否能正确实现

---

## 1. Go 没有类，但有“挂在类型上的方法”

如果你学过 Java、C#、Python，通常会把方法理解成“类的一部分”。

Go 不一样：
- Go 没有 class
- Go 用 `struct` 表示数据
- 用 **receiver（接收者）** 把方法绑定到某个类型上

看例子：

```go
package main

import "fmt"

type User struct {
	Name string
	Age  int
}

func (u User) Print() {
	fmt.Println("name:", u.Name)
	fmt.Println("age:", u.Age)
}

func main() {
	user := User{Name: "Tom", Age: 20}
	user.Print()
}
```

这里重点看这一行：

```go
func (u User) Print()
```

`u User` 就是接收者，意思是：
- 这个方法属于 `User` 类型
- 调用时写成 `user.Print()`

### 和其他语言的对比

你可以先把它粗略类比成：
- Java 的 `user.print()`
- Python 的 `user.print()` / `self`

但 Go 的本质不是“类里的成员函数”，而是“给类型附加函数”。

这个差异后面会越来越明显。

---

## 2. 值接收者：方法拿到的是副本

先看值接收者：

```go
package main

import "fmt"

type Counter struct {
	Value int
}

func (c Counter) AddOne() {
	c.Value++
}

func main() {
	counter := Counter{Value: 10}
	counter.AddOne()
	fmt.Println(counter.Value)
}
```

很多刚学 Go 的人会以为输出是 `11`，其实是：

```text
10
```

为什么？

因为：

```go
func (c Counter) AddOne()
```

这里的 `c` 是一个 **拷贝**。

也就是说调用 `counter.AddOne()` 时，更接近于：
- 把 `counter` 复制一份给 `c`
- 修改的是副本 `c`
- 原来的 `counter` 不变

### 什么时候适合值接收者

通常在下面场景用值接收者：

1. 方法只读，不修改数据
2. 结构体比较小，拷贝成本低
3. 这个类型本身就应该表现得像值，例如时间、坐标、配置快照

例如：

```go
package main

import "fmt"

type Point struct {
	X int
	Y int
}

func (p Point) Sum() int {
	return p.X + p.Y
}

func main() {
	p := Point{X: 3, Y: 4}
	fmt.Println(p.Sum())
}
```

这里 `Sum()` 只是读取数据，不需要改原对象，用值接收者就很自然。

---

## 3. 指针接收者：方法拿到的是原对象的地址

如果你想修改原对象，就要用指针接收者。

```go
package main

import "fmt"

type Counter struct {
	Value int
}

func (c *Counter) AddOne() {
	c.Value++
}

func main() {
	counter := Counter{Value: 10}
	counter.AddOne()
	fmt.Println(counter.Value)
}
```

这次输出是：

```text
11
```

关键区别在这里：

```go
func (c *Counter) AddOne()
```

`c` 不是值副本，而是指向原对象的指针。修改 `c.Value`，本质上就是修改原来的 `counter`。

### 和其他语言的对比

如果你熟悉：
- Java 里“对象引用”
- Python 里“对象默认按引用语义共享”

那你可能会习惯“方法里改了字段，外面也会变”。

Go 不默认这样。

Go 会明确要求你写出：
- `User`：按值
- `*User`：按指针

这也是 Go 的风格：**语义尽量写在代码表面。**

---

## 4. 一个判断标准：先问自己“我要不要改原对象？”

刚开始学习时，先用这个判断标准最实用：

### 用值接收者
- 方法只是读数据
- 不需要修改调用者持有的值

### 用指针接收者
- 方法要修改结构体字段
- 结构体较大，不想每次调用都拷贝
- 希望整个类型的方法风格保持一致

先别一次背太多规则，先抓住第一条：

**要改原对象，就优先考虑指针接收者。**

---

## 5. Go 的一个便利：调用时可以自动帮你取地址/解引用

看起来你可能会觉得：

```go
func (c *Counter) AddOne()
```

那是不是必须这样调用？

```go
(&counter).AddOne()
```

通常不用。

Go 会帮你做一层简化，所以你仍然可以写：

```go
counter.AddOne()
```

前提是这个值是可取地址的变量。

这点对新手很友好，但你要知道底层语义没有变：
- 方法定义在 `*Counter` 上
- 它本质上还是指针接收者

---

## 6. 为什么实际项目里很多结构体方法都偏向指针接收者

你后面会经常看到很多代码这样写：

```go
func (s *Service) Run() error
func (u *User) Rename(name string)
func (c *Config) Load() error
```

原因通常有三个：

1. 可能要修改状态
2. 避免大结构体拷贝
3. 和接口实现、方法集规则保持一致

第三点我们下一步再展开，现在只先建立印象，不强行灌输。

## 练习题

目标：

分别演示值接收者不会修改原对象、指针接收者可以修改原对象。

要求：

1. 定义 `Account` 结构体，包含 `Owner` 和 `Balance`
2. 定义值接收者方法 `DepositValue`
3. 定义指针接收者方法 `DepositPointer`
4. 在 `main()` 中分别调用并打印余额

### 参考答案代码

```go
package main

import "fmt"

type Account struct {
	Owner   string
	Balance int
}

func (a Account) DepositValue(amount int) {
	a.Balance += amount
}

func (a *Account) DepositPointer(amount int) {
	a.Balance += amount
}

func main() {
	account := Account{Owner: "Tom", Balance: 100}

	account.DepositValue(50)
	fmt.Println("after DepositValue:", account.Balance)

	account.DepositPointer(50)
	fmt.Println("after DepositPointer:", account.Balance)
}
```

### 预期运行结果

```text
after DepositValue: 100
after DepositPointer: 150
```

---

## 小结

这一小节先记住三句话：

1. **Go 没有类，方法是挂在类型上的**
2. **值接收者拿到的是副本**
3. **指针接收者可以修改原对象**

如果你现在只掌握到这三句，已经够进入练习了。
