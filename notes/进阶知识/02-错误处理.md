# 04 - 错误处理

## 本节目标

这一节先掌握 Go 错误处理里最核心的 2 个点：

1. `error` 是什么
2. 为什么 Go 通常写成 `if err != nil`

如果你之前主要用的是 Java、Python、JavaScript，最需要先转换的心智模型是：

**Go 默认不用异常来表示普通错误，而是把错误当普通返回值处理。**

---

## 1. Go 里的错误通常是“返回出来”，不是“抛出来”

很多语言遇到错误时，常见方式是抛异常：

- Java：`throw new Exception(...)`
- Python：`raise ValueError(...)`
- JavaScript：`throw new Error(...)`

Go 的主流风格不是这样。

Go 更常写成：

```go
value, err := someFunc()
if err != nil {
	// 处理错误
}
```

也就是说：
- 正常结果和错误一起返回
- 调用方必须显式检查
- 没有“异常自动向上飞”的默认控制流

### 这种设计的好处

对有其他语言经验的人来说，一开始会觉得啰嗦。

但 Go 想换来几件事：

1. **控制流更直白**：函数可能失败这件事写在返回值里
2. **更容易读代码**：不需要到处追踪哪里会抛异常
3. **错误处理更局部**：失败后怎么处理，由当前调用点决定

这很符合 Go 的整体哲学：

**少魔法，重显式。**

---

## 2. `error` 本质上是一个接口

Go 标准库里，`error` 是一个内置接口：

```go
type error interface {
	Error() string
}
```

意思很简单：
- 只要一个类型实现了 `Error() string` 方法
- 它就可以被当成 `error`

你现在先不用急着自己定义错误类型，先把它理解成：

**`error` 是 Go 用来统一表示“操作失败信息”的标准接口。**

在日常学习阶段，你最常见的是标准库直接返回它。

例如伪代码：

```go
result, err := doSomething()
if err != nil {
	fmt.Println("出错了:", err)
	return
}
```

这里的 `err`：
- 为 `nil`：表示没有错误
- 不为 `nil`：表示发生了错误

---

## 3. `nil` 在这里表示“没有错误”

这点非常重要。

很多语言里你可能会习惯：
- 返回 `true/false`
- 抛异常
- 返回 `null` 或特殊值

而在 Go 里，错误处理经常是：

```go
if err != nil {
	// 出错
}
```

也就是：
- `err == nil`：成功
- `err != nil`：失败

### 示例：自己写一个可能失败的函数

```go
package main

import (
	"errors"
	"fmt"
)

func divide(a int, b int) (int, error) {
	if b == 0 {
		return 0, errors.New("除数不能为 0")
	}

	return a / b, nil
}

func main() {
	result, err := divide(10, 2)
	if err != nil {
		fmt.Println("计算失败:", err)
		return
	}

	fmt.Println("结果是:", result)
}
```

这个例子里你要重点看两种返回：

### 成功时

```go
return a / b, nil
```

表示：
- 第一个返回值是正常结果
- 第二个返回值没有错误，所以是 `nil`

### 失败时

```go
return 0, errors.New("除数不能为 0")
```

表示：
- 这次没有有效结果，所以先放一个占位值 `0`
- 同时返回一个具体错误

---

## 4. 为什么 Go 喜欢“结果 + error”这种形式

这是 Go 最经典的函数签名之一：

```go
func xxx(...) (T, error)
```

你以后会高频看到：

- `os.Open(...) (*File, error)`
- `strconv.Atoi(...) (int, error)`
- `json.Marshal(...) ([]byte, error)`

### 和其他语言对比

#### 在 Python 里你可能这样写

```python
value = int("123")
```

如果失败，抛 `ValueError`。

#### 在 Go 里更常见的是

```go
value, err := strconv.Atoi("123")
if err != nil {
	// 处理错误
}
```

也就是说，Go 把“可能失败”直接放进函数签名里，而不是藏在运行时异常机制后面。

---

## 5. `if err != nil` 为什么几乎成了固定写法

因为它同时满足三个目标：

1. **近距离处理错误**：刚拿到结果就检查
2. **减少嵌套**：出错就尽早返回
3. **统一风格**：整个生态都这样写，阅读成本低

看一个 Go 风格很强的例子：

```go
package main

import (
	"errors"
	"fmt"
)

func validateName(name string) error {
	if name == "" {
		return errors.New("name 不能为空")
	}

	return nil
}

func greet(name string) (string, error) {
	err := validateName(name)
	if err != nil {
		return "", err
	}

	return "Hello, " + name, nil
}

func main() {
	message, err := greet("Go")
	if err != nil {
		fmt.Println("失败:", err)
		return
	}

	fmt.Println(message)
}
```

这里最值得你建立习惯的是：

- 调用后立刻检查 `err`
- 出错就返回
- 不要把错误先放着，过几行再说

这在 Go 里属于非常重要的惯用法。

---

## 6. Go 为什么不鼓励把普通错误当异常流

这里要区分两件事：

- **普通业务失败**：文件不存在、输入不合法、解析失败
- **真正异常的程序状态**：程序已经无法继续，比如严重 bug

在 Go 里：
- 前者通常用 `error`
- 后者才可能用 `panic`

你现在先记一个足够实用的规则：

**能预料到的失败，用 `error`；不要一上来就想着用 `panic`。**

例如：
- 用户输入错了：这是正常错误，不是程序崩溃理由
- 除数为 0（来自外部输入）：返回 `error` 更合理

Go 的思路是：

**错误是业务流程的一部分，不一定是灾难。**

---

## 7. 一个初学者最容易犯的错误

### 错误 1：拿到 `err` 不检查

错误示意：

```go
result, err := divide(10, 0)
fmt.Println(result)
fmt.Println(err)
```

语法上不一定错，但风格上不好。

为什么不好：
- 你已经知道函数可能失败
- 却没有立刻决定“失败了怎么办”
- 后面代码容易在错误状态下继续跑

Go 更推荐：

```go
result, err := divide(10, 0)
if err != nil {
	fmt.Println("计算失败:", err)
	return
}

fmt.Println(result)
```

### 错误 2：用特殊值偷偷表示失败

比如：

```go
func divide(a int, b int) int {
	if b == 0 {
		return -1
	}
	return a / b
}
```

在别的语言里你可能见过这种写法，但在 Go 里通常不推荐。

为什么？
- `-1` 到底是真结果还是错误标记，不够清晰
- 调用方不知道必须检查这个特殊值
- 不如 `(..., error)` 明确

---

## 8. 先建立一个最重要的模板

你现在先把这个模板练熟：

```go
result, err := someFunc()
if err != nil {
	return err
}
```

或者在 `main` 里：

```go
result, err := someFunc()
if err != nil {
	fmt.Println("失败:", err)
	return
}

fmt.Println(result)
```

你后面学文件操作、字符串转换、JSON、HTTP 时，都会不断重复这个模式。

## 练习题

目标：

自己写一个会成功/失败两种结果的函数，练习 `(结果, error)` 和 `if err != nil`。

要求：

1. 定义函数 `safeDivide(a int, b int) (int, error)`
2. 当 `b == 0` 时返回错误
3. 当 `b != 0` 时返回商和 `nil`
4. 在 `main()` 中分别测试成功和失败两种情况

### 参考答案代码

```go
package main

import (
	"errors"
	"fmt"
)

func safeDivide(a int, b int) (int, error) {
	if b == 0 {
		return 0, errors.New("division by zero")
	}

	return a / b, nil
}

func main() {
	result, err := safeDivide(10, 2)
	if err != nil {
		fmt.Println("error:", err)
		return
	}
	fmt.Println("result:", result)

	result, err = safeDivide(10, 0)
	if err != nil {
		fmt.Println("error:", err)
		return
	}
	fmt.Println("result:", result)
}
```

### 预期运行结果

```text
result: 5
error: division by zero
```

---

## 小结

这节先记住 4 句话：

1. **Go 的普通错误通常通过返回值传递，不靠异常。**
2. **`error` 是接口，`nil` 表示没有错误。**
3. **最常见函数签名是 `(结果, error)`。**
4. **拿到 `err` 后立刻用 `if err != nil` 检查。**
