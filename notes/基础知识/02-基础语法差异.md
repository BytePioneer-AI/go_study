# 02 - 基础语法差异

## 本节目标

这一节先掌握两个最常用、也最能体现 Go 风格的语法点：

1. 短变量声明 `:=`
2. 多返回值

这两个特性你后面会高频使用，尤其是在错误处理里。

---

## 1. 短变量声明 `:=`

在很多语言里，声明变量和给变量赋值往往是分开的，或者语法比较统一：

- Java：`int x = 10;`
- Python：`x = 10`
- JavaScript：`let x = 10`

Go 同时保留了两种写法：

```go
var x int = 10
```

和：

```go
x := 10
```

其中 `:=` 叫做 **短变量声明**，它会同时完成两件事：
- 声明变量
- 根据右边的值推断类型

### 推荐心智模型

你可以先这样记：

- **函数内部**，优先用 `:=`
- **需要零值、显式类型、包级变量** 时，再考虑 `var`

### 示例

```go
package main

import "fmt"

func main() {
	name := "Gopher"
	age := 3
	fmt.Println(name, age)
}
```

这里：
- `name` 被推断为 `string`
- `age` 被推断为 `int`

### Go 和其他语言的差异

在 Python/JavaScript 里，变量类型通常可以随时变：

```python
x = 10
x = "hello"
```

但 Go 不行：

```go
x := 10
x = "hello" // 编译错误
```

Go 的类型一旦确定，就不能随意变。这个限制换来的是更强的可读性和更早的错误发现。

### 什么时候不能用 `:=`

`:=` 只能在 **函数内部** 使用，不能用于包级别。

错误示例：

```go
package main

x := 10 // 错误

func main() {}
```

如果在函数外声明变量，要用：

```go
package main

var x = 10

func main() {}
```

### 一个常见坑：`:=` 不是“重新赋值”

看这个例子：

```go
package main

import "fmt"

func main() {
	name := "Go"
	name = "Golang" // 这是赋值
	fmt.Println(name)
}
```

这里第二行必须用 `=`，因为 `name` 已经声明过了。

也就是说：
- `:=` = 首次声明
- `=` = 给已有变量赋值

## 2. 多返回值

这是 Go 非常有代表性的设计。

很多语言里，函数通常只返回一个值。如果要返回多个值：
- Python 用 tuple
- Java 可能要造一个类
- JavaScript 常用对象/数组

Go 允许函数直接返回多个值。

### 示例

```go
package main

import "fmt"

func swap(a string, b string) (string, string) {
	return b, a
}

func main() {
	first, second := swap("hello", "go")
	fmt.Println(first, second)
}
```

这里 `swap` 一次返回两个字符串。

### 为什么 Go 要这样设计

因为 Go 希望把“正常结果”和“错误信息”一起返回，而不是依赖异常。

你后面会经常看到这种形式：

```go
value, err := someFunc()
```

这正是 Go 错误处理风格的基础。

所以你现在学多返回值，不只是语法问题，而是在为后面的 `error` 做准备。

### 可以忽略不需要的返回值

Go 提供了空白标识符 `_`：

```go
package main

import "fmt"

func getNameAndAge() (string, int) {
	return "Gopher", 3
}

func main() {
	name, _ := getNameAndAge()
	fmt.Println(name)
}
```

这里表示：
- 我只要 `name`
- 第二个返回值先丢弃

### 注意：Go 不允许“拿少了”

下面这种写法会报错：

```go
name := getNameAndAge() // 错误：返回了两个值，却只接一个变量
```

Go 要求你明确处理返回值，要么全部接住，要么用 `_` 忽略。

这体现了 Go 的一个风格：**尽量显式，不帮你偷偷处理。**

---

## 结合示例：两个概念一起用

```go
package main

import "fmt"

func divide(a int, b int) (int, int) {
	quotient := a / b
	remainder := a % b
	return quotient, remainder
}

func main() {
	q, r := divide(10, 3)
	fmt.Println("quotient:", q)
	fmt.Println("remainder:", r)
}
```

这里同时用了：
- `:=` 做局部变量声明
- 多返回值返回商和余数

## 练习题

目标：

练习短变量声明、多返回值，以及用 `_` 忽略不需要的返回值。

要求：

1. 定义函数 `calc(a int, b int) (int, int)`
2. 返回 `a + b` 和 `a * b`
3. 在 `main()` 中先完整接收两个返回值
4. 再调用一次，只接收和，忽略积

### 参考答案代码

```go
package main

import "fmt"

func calc(a int, b int) (int, int) {
	return a + b, a * b
}

func main() {
	x := 3
	y := 4

	sum, product := calc(x, y)
	fmt.Println("sum:", sum)
	fmt.Println("product:", product)

	sumOnly, _ := calc(5, 6)
	fmt.Println("sum only:", sumOnly)
}
```

### 预期运行结果

```text
sum: 7
product: 12
sum only: 11
```

---

## 小结

这节先记住三句话：

1. **函数内部优先用 `:=`**
2. **`:=` 是声明，`=` 是赋值**
3. **Go 函数可以直接返回多个值**

后面学错误处理时，你会看到这两个特性几乎总是一起出现。
