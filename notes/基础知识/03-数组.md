# 03 - 数组

## 本节目标

这一节先掌握两个最核心的问题：

1. Go 里的数组到底是什么
2. 数组和切片最关键的区别是什么

如果你之前写过 Java、Python、JavaScript，最需要先转换的心智模型是：

**Go 的数组不是“可自动伸缩的列表”，而是“长度属于类型的一部分的定长序列”。**

---

## 1. 数组是定长的，长度写进类型里

先看最基本的例子：

```go
package main

import "fmt"

func main() {
	nums := [3]int{10, 20, 30}
	fmt.Println(nums)
	fmt.Println(nums[0])
	fmt.Println(len(nums))
}
```

这里的 `[3]int` 意思不是“有 3 个元素的 int 列表”这么简单。

它真正表示的是：

- 这是一个数组
- 元素类型是 `int`
- 长度固定为 `3`

### 和其他语言的对比

在很多语言里，你会把数组/列表理解成：
- 一组同类型元素
- 长度通常可以变化
- 变量类型和长度通常是分开的

但 Go 不一样。

在 Go 里：

- `[3]int` 和 `[4]int` 是 **两个不同的类型**
- 长度不是值的一部分，而是类型的一部分

看这个例子：

```go
package main

func main() {
	var a [3]int
	var b [4]int

	a = b // 编译错误
}
```

为什么报错？

因为 `a` 的类型是 `[3]int`，`b` 的类型是 `[4]int`，它们根本不是同一种类型。

这在很多其他语言里不常见，但在 Go 里非常重要。

---

## 2. 数组有固定长度，所以适合表示“大小已知的数据”

例如：

- 一周 7 天的数据
- 固定 3 个坐标值
- 固定长度的缓存块

示例：

```go
package main

import "fmt"

func main() {
	weekTemps := [7]int{21, 22, 20, 23, 24, 22, 19}
	fmt.Println(weekTemps)
}
```

如果你的数据天然就是“固定个数”，数组很合适。

但如果你需要“追加元素”“长度动态变化”，那通常就不是数组的场景，而是切片的场景。

所以你可以先记一句：

**数组偏静态，切片偏动态。**

---

## 3. 数组是值类型

这是 Go 数组最容易让其他语言使用者踩坑的点。

看例子：

```go
package main

import "fmt"

func main() {
	a := [3]int{1, 2, 3}
	b := a
	b[0] = 100

	fmt.Println("a:", a)
	fmt.Println("b:", b)
}
```

输出会是：

```go
a: [1 2 3]
b: [100 2 3]
```

也就是说，`b := a` 时，发生的是 **整份拷贝**。

### 和其他语言的对比

在 Python/JavaScript/Java 里，你很容易把“数组/列表变量赋值”理解成：
- 两个变量指向同一份底层数据
- 改一边，另一边也变

但 Go 的数组默认不是这个语义。

Go 数组更像：

- 复制一个 `int`
- 复制一个 `struct`
- 复制一个完整的值

所以数组赋值、传参时，默认都是拷贝整个数组。

这就是“数组是值类型”的核心含义。

---

## 4. 数组传给函数时，也会复制

看例子：

```go
package main

import "fmt"

func changeFirst(nums [3]int) {
	nums[0] = 999
}

func main() {
	nums := [3]int{1, 2, 3}
	changeFirst(nums)
	fmt.Println(nums)
}
```

输出仍然是：

```go
[1 2 3]
```

因为函数拿到的是数组的副本，不是原数组本身。

如果你想修改原数组，要传指针：

```go
package main

import "fmt"

func changeFirst(nums *[3]int) {
	(*nums)[0] = 999
}

func main() {
	nums := [3]int{1, 2, 3}
	changeFirst(&nums)
	fmt.Println(nums)
}
```

这里先不用把重点放在指针细节上，你只要先记住：

**数组作为参数传递时，默认也按值复制。**

---

## 5. 数组和切片的最关键区别

你后面会发现，Go 日常开发里切片远比数组常见。

先只抓最关键的一层区别：

### 数组

- 长度固定
- 长度属于类型的一部分
- 赋值/传参默认复制整个数组

### 切片

- 长度可变
- 更像对一段底层数据的“视图”
- 日常业务代码里更常用

所以 Go 程序员看到：

```go
[3]int
```

会立刻意识到：这是一个“很明确地固定长度”的数据结构。

而看到：

```go
[]int
```

就会意识到：这是切片，不是数组。

这两个写法长得很像，但语义差很多。

---

## 6. 一个常见误区

很多刚学 Go 的人会以为：

```go
nums := []int{1, 2, 3}
```

这是“数组”。

其实不是。

这是 **切片**。

真正的数组写法是：

```go
nums := [3]int{1, 2, 3}
```

你可以把这当成当前最重要的辨认题：

- `[3]int` → 数组
- `[]int` → 切片

方括号里有没有具体长度，决定了它是不是数组。

## 7. 可运行示例

你可以在当前目录新建一个文件，比如 `array_demo.go`，写入下面代码：

```go
package main

import "fmt"

func main() {
	nums := [3]int{10, 20, 30}
	copyNums := nums
	copyNums[1] = 99

	fmt.Println("nums:", nums)
	fmt.Println("copyNums:", copyNums)
	fmt.Println("len(nums):", len(nums))
}
```

你自己运行：

```bash
go run array_demo.go
```

你应该观察到：

1. `nums` 长度是固定的 3
2. `copyNums := nums` 后修改 `copyNums`，不会影响 `nums`

## 练习题

目标：

练习数组声明、索引访问，以及验证数组赋值是拷贝。

要求：

1. 定义一个 `[4]string` 数组保存四个季节
2. 打印整个数组、第 2 个元素和数组长度
3. 再定义一个 `[3]int` 数组并复制给新变量
4. 修改副本后同时打印原数组和副本

### 参考答案代码

```go
package main

import "fmt"

func main() {
	seasons := [4]string{"Spring", "Summer", "Autumn", "Winter"}
	fmt.Println(seasons)
	fmt.Println(seasons[1])
	fmt.Println(len(seasons))

	nums := [3]int{1, 2, 3}
	other := nums
	other[0] = 100

	fmt.Println("nums:", nums)
	fmt.Println("other:", other)
}
```

### 预期运行结果

```text
[Spring Summer Autumn Winter]
Summer
4
nums: [1 2 3]
other: [100 2 3]
```

## 本节小结

这一节你先记住 3 句话：

1. **数组是定长的，长度属于类型的一部分**
2. **数组是值类型，赋值和传参都会复制**
3. **Go 里日常更常用的是切片，不是数组**

你现在学到的位置是：**已经进入集合类型，刚开始掌握数组的基本语义**。
