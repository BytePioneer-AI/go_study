# 06 - 包（package）

## 本节目标

这一节先掌握两个最核心的问题：

1. Go 里的 `package` 到底是什么
2. 为什么文件开头要写 `package main`，以及 `main` 和普通包有什么区别

如果你之前写过 Java、Python、JavaScript，这一节最重要的心智转换是：

**Go 的包首先是“代码组织和可见性边界”，不是类命名空间，也不是随便 import 一个文件。**

---

## 1. 什么是包

先看你现在最熟悉的一段代码：

```go
package main

import "fmt"

func main() {
	fmt.Println("Hello, Go!")
}
```

文件开头这句：

```go
package main
```

表示：

**这个 `.go` 文件属于 `main` 包。**

在 Go 里，包是组织代码的基本单位。你可以先把它理解成：

- 一组同属一个目录、一起编译的 Go 文件
- 这些文件共同组成一个包
- 包名写在每个文件开头

### 和其他语言的对比

#### Java

你可能会想到：

```java
package com.example.app;
```

但 Go 的包没有 Java 那么重的“全路径命名空间”味道。

#### Python

你可能会想到模块 / 包：

```python
import utils
```

但 Go 不是“直接 import 某个文件”，而是 import 一个包。

#### JavaScript

你可能习惯：

```javascript
import { add } from "./utils.js"
```

Go 不是按文件导入，而是按包来组织。

### 你先记一句

**Go 里最常见的组织方式不是“这个函数在哪个文件”，而是“这个函数属于哪个包”。**

---

## 2. `package main` 是特殊包

Go 里不是所有包都能直接运行。

只有满足下面两个条件，程序才能 `go run`：

1. 包名是 `main`
2. 里面有 `main()` 函数

例如：

```go
package main

import "fmt"

func main() {
	fmt.Println("program started")
}
```

这里：

- `package main` 表示这是一个可执行程序包
- `func main()` 是程序入口

### 这和其他语言很像但又不完全一样

- Java 有 `public static void main`
- C / C++ 有 `main`
- Python 常见 `if __name__ == "__main__":`

但 Go 更直接：

**只要是 `main` 包 + `main()` 函数，它就是可执行程序。**

---

## 3. 普通包不是程序入口，而是被别的包使用

比如你写一个工具包：

```go
package mathutil

func Add(a int, b int) int {
	return a + b
}
```

这不是入口程序，而是一个普通包。

它的作用是：

- 提供函数
- 提供类型
- 被其他包导入使用

你可以把它理解成“可复用代码单元”。

### 心智区别

- `main` 包：负责启动程序
- 普通包：负责提供功能

这和很多语言里“应用入口”和“库代码”分离的思路一致，但 Go 用 `package` 机制统一起来了。

---

## 4. `import` 导入的是包，不是文件

看一个常见例子：

```go
package main

import "fmt"

func main() {
	fmt.Println("Hello")
}
```

这里导入的是：

```go
import "fmt"
```

不是某个 `fmt.go` 文件，而是 `fmt` 这个包。

然后通过：

```go
fmt.Println(...)
```

使用这个包里导出的函数。

### 这是初学者很容易混淆的点

在一些语言里你会关注：

- 导入哪个文件
- 文件名叫什么

但在 Go 里，更重要的是：

- 导入哪个包
- 这个包暴露了哪些名字

---

## 5. 同一个目录下的 Go 文件，通常属于同一个包

这是 Go 很重要的目录约定。

假设一个目录下有两个文件：

### `main.go`

```go
package main

func main() {
	sayHello()
}
```

### `hello.go`

```go
package main

import "fmt"

func sayHello() {
	fmt.Println("hello")
}
```

虽然函数分布在两个文件里，但因为它们都在同一个目录、同一个包 `main` 里，所以可以一起编译。

### 这和其他语言的差异

有些语言会更强调“一个文件一个类”或“必须显式引用文件”。

Go 不这么想。

Go 的重点是：

- 同目录
- 同 package
- 一起构成一个编译单元

所以在 Go 里，**文件主要是物理拆分方式，包才是逻辑组织单位。**

---

## 6. 标识符大小写决定是否导出

这是 Go 包设计里最有特点的一点。

看例子：

```go
package mathutil

func Add(a int, b int) int {
	return a + b
}

func subtract(a int, b int) int {
	return a - b
}
```

这里：

- `Add` 首字母大写，可以被包外使用
- `subtract` 首字母小写，只能在包内使用

### 和其他语言的对比

#### Java / C#

你可能会写 `public` / `private`

#### Python

你可能靠约定 `_name`

#### JavaScript

你可能靠 `export`

#### Go

Go 直接用：

**首字母大写 = 导出，首字母小写 = 包内可见**

这非常简单，也非常 Go。

### 先记结论

如果你将来看到：

- `fmt.Println` 能用，是因为 `Println` 大写开头，被导出
- 某个小写函数在别的包里不能调，不是你不会写，而是它本来就没导出

## 7. 一个最小可运行的“包拆分”例子

你可以在当前目录自己练习下面这个结构。

### `main.go`

```go
package main

import (
	"fmt"
	"go_study/greetings"
)

func main() {
	msg := greetings.Hello("Go")
	fmt.Println(msg)
}
```

### `greetings/greetings.go`

```go
package greetings

func Hello(name string) string {
	return "Hello, " + name
}
```

### 这个例子想说明什么

1. `greetings` 是一个普通包
2. `main` 包导入它并使用导出函数 `Hello`
3. `Hello` 大写，所以包外可见

## 8. 你现在最需要建立的包思维

学到这里，先不要一下子去记太多模块路径细节。

你先建立这 4 个判断：

1. 文件开头必须声明自己属于哪个包
2. `main` 包是可执行程序入口包
3. `import` 导入的是包，不是文件
4. 大写标识符才能被其他包使用

如果这 4 个点清楚了，后面学 `go mod`、自定义包、跨目录组织代码时就会顺很多。

## 常见误区

### 误区 1：文件名决定能不能被导入

不是。

Go 主要看：

- 目录
- package 名
- 模块路径

不是看单个文件名。

### 误区 2：同目录下可以随便混不同 package

通常不行。

同一个目录下的 `.go` 文件，通常应该属于同一个包。

### 误区 3：小写函数只是“风格问题”

不是。

在 Go 里，大写/小写直接决定可见性。

## 练习题

### 练习 1

下面代码能不能运行？为什么？

```go
package utils

import "fmt"

func main() {
	fmt.Println("hello")
}
```

先自己想，再看答案。

### 参考答案

不能直接作为程序入口运行。

原因：虽然有 `main()` 函数，但包名不是 `main`，而是 `utils`。

在 Go 里，可执行程序要求：

- 包名必须是 `main`
- 入口函数必须是 `main()`

---

### 练习 2

看代码：

```go
package mathutil

func add(a int, b int) int {
	return a + b
}
```

如果别的包导入 `mathutil`，能不能调用 `add`？为什么？

### 参考答案

不能。

因为 `add` 是小写开头，没有导出，只能在 `mathutil` 包内部使用。

如果想让包外调用，要改成：

```go
package mathutil

func Add(a int, b int) int {
	return a + b
}
```
