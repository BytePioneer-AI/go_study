# 00 - Go 语言背景与特点

## 历史

Go 由 Google 的三位工程师创造：
- **Robert Griesemer**（参与过 V8 引擎）
- **Rob Pike**（Unix 和 UTF-8 的创造者之一）
- **Ken Thompson**（C 语言和 Unix 的创造者）

2007 年开始设计，2009 年开源，2012 年发布 1.0。

**诞生背景**：Google 内部大量使用 C++，但 C++ 编译慢、代码复杂、并发难写。三人想要一门"更好的系统语言"。

---

## 设计哲学：少即是多

Go 故意做得很简单：
- 没有泛型（1.18 才加入，且刻意保持克制）
- 没有异常（只有 error 返回值）
- 没有继承（只有组合）
- 没有隐式类型转换
- 关键字只有 25 个（Python 有 35 个，Java 有 50 个）

这不是能力不足，而是**刻意取舍**——简单的语言让大团队协作更容易。

---

## 核心特点

### 1. 编译型 + 垃圾回收
- 像 C 一样编译成原生二进制，启动快、性能好
- 像 Java 一样有 GC，不用手动管理内存
- 编译速度极快（比 C++ 快几十倍）

### 2. 原生并发支持
Go 的并发是语言级别的，不是库：
```go
go func() {
    // 这就启动了一个并发任务（goroutine）
}()
```
goroutine 比线程轻量得多，可以轻松跑几十万个。

### 3. 强类型 + 静态类型
```go
x := 42        // 编译器推断为 int，之后不能赋字符串
x = "hello"    // 编译错误
```

### 4. 工具链内置
不需要第三方工具：
- `go fmt` 统一格式（团队不争论代码风格）
- `go test` 内置测试
- `go build` 编译
- `go doc` 文档

---

## Go 适合做什么

| 场景 | 代表项目 |
|------|---------|
| 云基础设施 | Docker、Kubernetes、Terraform |
| 微服务 / API 服务 | 大量互联网公司后端 |
| 命令行工具 | Hugo、cobra |
| 网络编程 | 标准库直接支持 |

**不适合**：GUI 应用、数据科学（Python 更合适）、底层系统编程（Rust 更合适）。

---

## 与你可能熟悉的语言对比

| 特性 | Go | Python | Java | C++ |
|------|----|--------|------|-----|
| 编译 | ✅ 原生二进制 | ❌ 解释 | ✅ JVM字节码 | ✅ 原生二进制 |
| GC | ✅ | ✅ | ✅ | ❌ |
| 并发 | goroutine（极轻量） | GIL限制 | 线程 | 线程 |
| 学习曲线 | 低 | 低 | 中 | 高 |
| 编译速度 | 极快 | - | 慢 | 很慢 |

---

## 小结

Go 的核心价值主张：**工程效率**。
不追求最强的表达力，而是让一个大团队能快速、一致地写出可维护的代码。
