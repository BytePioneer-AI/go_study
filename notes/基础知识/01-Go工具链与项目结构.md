# 01 - Go 工具链与项目结构

## 核心概念：Module

Go 用 **module** 管理项目，类似 npm 的 `package.json` 或 Python 的 `pyproject.toml`。

```bash
go mod init <module-name>
```

生成 `go.mod` 文件，定义模块名和 Go 版本：

```
module go_study

go 1.25.5
```

---

## 目录约定

```
my-project/
├── go.mod          # 模块定义（必须有）
├── main.go         # 可执行程序入口（package main）
├── go.sum          # 依赖锁文件（自动生成，不要手动改）
└── internal/       # 只能被本模块引用的包（Go 强制约束）
```

**Go 里目录 = 包**，一个目录下所有 `.go` 文件属于同一个包。

---

## 两种包的区别

| 包名 | 用途 |
|------|------|
| `package main` | 可执行程序，必须有 `main()` 函数 |
| `package xxx` | 库包，被其他包 import |

---

## 常用命令

```bash
go run main.go      # 直接运行，不生成二进制
go run .            # 同上，运行当前目录
go build .          # 编译，生成可执行文件
go build -o hello.exe .  # 编译并指定输出文件名
go mod tidy         # 自动添加缺失依赖、删除多余依赖
go test ./...       # 运行所有测试
go fmt ./...        # 格式化代码
go vet ./...        # 静态检查
```

---

## go run vs go build

| 命令 | 编译 | 产物 | 用途 |
|---|---|---|---|
| `go run .` | 是 | 无（临时） | 开发调试 |
| `go build .` | 是 | 可执行文件 | 发布部署 |

- `go run` 内部先编译再运行，临时产物存在系统临时目录，运行结束后清理
- 开发阶段只用 `go run .` 即可，不需要手动 build
- `go build` 生成的文件在**当前项目目录**下

---

## 依赖管理

- `go mod init` 每个项目只执行**一次**，生成 `go.mod`
- 依赖**不在项目目录里**，缓存在系统全局：`%GOPATH%\pkg\mod`（通常是 `C:\Users\你\go\pkg\mod`）
- 同一个库同一版本只下载一次，多个项目共享，没有 `node_modules` 的磁盘浪费问题
- 查看 GOPATH：`go env GOPATH`

---

## 跨平台编译

Go 支持交叉编译，在 Windows 上可直接编译出其他平台的二进制：

```bash
# 编译产物因平台而异
# Windows → go_study.exe
# Linux / macOS → go_study（无后缀）

# 在 Windows 上编译 Linux 版本
GOOS=linux GOARCH=amd64 go build -o go_study .

# 编译 macOS 版本
GOOS=darwin GOARCH=amd64 go build -o go_study .
```

## 练习题

目标：

创建一个子包，并在 `main.go` 中调用它，熟悉 Go 的模块路径和包导入方式。

要求：

1. 在当前目录下创建 `greeting/greeting.go`
2. 包名使用 `greeting`
3. 定义函数 `Hello(name string) string`
4. 在 `main.go` 中导入并调用它

### 参考答案代码

`greeting/greeting.go`

```go
package greeting

func Hello(name string) string {
	return "Hello, " + name + "!"
}
```

`main.go`

```go
package main

import (
	"fmt"
	"go_study/greeting"
)

func main() {
	fmt.Println(greeting.Hello("Gopher"))
}
```

### 预期运行结果

```text
Hello, Gopher!
```
