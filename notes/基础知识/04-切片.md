# 04 - 切片

## 本节目标

这一节先掌握两个最核心的问题：

1. 切片 `[]T` 到底是什么
2. 为什么 Go 日常开发里切片比数组更常用

如果你刚学完数组，这一节最重要的心智转换是：

**切片不是数组本身，而是对一段数组数据的描述。**

你可以先把它理解成：

- “一段连续元素的窗口”
- “对底层数组的一个视图”
- “长度可变的常用容器”

---

## 1. 切片和数组长得像，但不是一回事

先看两个写法：

```go
[3]int
```

和：

```go
[]int
```

它们只差一个数字，但含义完全不同。

### `[3]int`

- 是数组
- 长度固定为 3
- 长度属于类型的一部分

### `[]int`

- 是切片
- 类型里没有固定长度
- 更适合日常处理一组数据

这是当前最重要的辨认题：

- `[3]int` → 数组
- `[]int` → 切片

---

## 2. 切片为什么更常用

因为业务代码里的数据通常不是天然定长的。

例如：

- 一批用户列表
- 一组订单
- 一段日志记录
- 一组请求结果

这些数据的共同特点是：

- 个数通常运行时才知道
- 经常需要追加
- 很少一开始就写死长度

所以 Go 日常更常写：

```go
names := []string{"Tom", "Jack", "Lucy"}
```

而不是：

```go
names := [3]string{"Tom", "Jack", "Lucy"}
```

后者不是错，只是更“硬编码固定长度”。

---

## 3. 切片有长度 len，也有容量 cap

先看例子：

```go
package main

import "fmt"

func main() {
	nums := []int{10, 20, 30}
	fmt.Println("nums:", nums)
	fmt.Println("len:", len(nums))
	fmt.Println("cap:", cap(nums))
}
```

这里先只记最直观的定义：

- `len`：当前切片里有几个元素
- `cap`：从当前切片起点到底层数组末尾，还能覆盖多大范围

如果你现在觉得 `cap` 有点抽象，很正常。

学习顺序建议是：

1. 先熟悉 `len`
2. 知道 `cap` 存在
3. 等学到 `append` 和底层共享时，再真正理解它

当前阶段不要把注意力全放在 `cap` 上。

---

## 4. 切片通常引用同一份底层数据

这是切片和数组最关键的语义区别。

看例子：

```go
package main

import "fmt"

func main() {
	arr := [4]int{10, 20, 30, 40}
	s := arr[1:3]

	fmt.Println("arr:", arr)
	fmt.Println("s:", s)

	s[0] = 99

	fmt.Println("arr:", arr)
	fmt.Println("s:", s)
}
```

`s := arr[1:3]` 表示取：

- 下标 1 开始
- 下标 3 结束（但**不包含 3**）

所以 `s` 一开始是：

```go
[20 30]
```

然后当你执行：

```go
s[0] = 99
```

原数组 `arr` 里的对应位置也会变。

### 这说明什么？

说明切片不像数组那样默认整份复制。

切片更像是：

- 自己不直接存整份数据
- 而是“看着”底层数组中的一段

所以改切片里的元素，本质上是在改底层那份数据。

---

## 5. 切片表达式：左闭右开

Go 的切片区间规则和很多语言类似：

```go
a[start:end]
```

含义是：

- 包含 `start`
- 不包含 `end`

也就是 **左闭右开**。

例如：

```go
nums := []int{0, 1, 2, 3, 4}
fmt.Println(nums[1:4])
```

输出是：

```go
[1 2 3]
```

因为取到了下标 1、2、3，没有取到 4。

### 为什么这种规则很常见

因为它有两个好处：

1. 长度容易算：`end - start`
2. 相邻区间容易拼接，不重不漏

例如：

- `nums[0:2]`
- `nums[2:5]`

这两个区间正好接上。

---

## 6. 从数组创建切片

这是最容易理解切片本质的方式。

```go
package main

import "fmt"

func main() {
	arr := [5]int{1, 2, 3, 4, 5}

	a := arr[0:2]
	b := arr[2:5]

	fmt.Println(a)
	fmt.Println(b)
}
```

这里：

- `a` 看 arr 的前两个元素
- `b` 看 arr 的后三个元素

注意：`a` 和 `b` 都不是新数组，而是基于 `arr` 生成的切片。

---

## 7. 直接字面量创建切片

平时更常见的是直接写：

```go
nums := []int{1, 2, 3}
```

这就是创建一个切片。

你可以先把它理解为：

- Go 帮你准备好了底层存储
- 再返回一个切片值来描述这段数据

在当前阶段，这样理解就够用了。

---

## 8. `append`：给切片追加元素

这也是切片最常用的操作。

```go
package main

import "fmt"

func main() {
	nums := []int{1, 2, 3}
	nums = append(nums, 4)
	nums = append(nums, 5, 6)

	fmt.Println(nums)
}
```

这里注意一个非常重要的写法：

```go
nums = append(nums, 4)
```

而不是只写：

```go
append(nums, 4)
```

### 为什么必须接回去

因为 `append` 的返回值是 **新的切片值**。

它有时可能还是原来的底层数组，
也有时可能因为容量不够，换了一块新的底层数组。

所以 Go 要求你显式接住返回值。

这又体现了 Go 的风格：

**可能变化的结果，显式返回给你。**

---

## 9. 一个常见误区：切片不是“引用类型”这四个字就能解释完

很多教程会简单说“切片是引用类型”。

这句话有一点帮助，但也容易误导。

更准确的理解是：

- 切片本身也是一个值
- 这个值里描述了底层数组的一段区域
- 把切片赋值给另一个变量时，复制的是这个“描述信息”
- 两个切片可能仍然指向同一段底层数据

看例子：

```go
package main

import "fmt"

func main() {
	a := []int{1, 2, 3}
	b := a
	b[0] = 100

	fmt.Println("a:", a)
	fmt.Println("b:", b)
}
```

这里 `a` 和 `b` 都会看到修改。

所以和数组对比时，你先记住这个现象就够了：

- 数组赋值后，通常互不影响
- 切片赋值后，常常会相互影响底层数据

## 10. 可运行示例

你可以在当前目录新建一个文件，比如 `slice_demo.go`：

```go
package main

import "fmt"

func main() {
	nums := []int{10, 20, 30}
	other := nums
	other[1] = 99

	fmt.Println("nums:", nums)
	fmt.Println("other:", other)

	nums = append(nums, 40)
	fmt.Println("after append:", nums)
}
```

你自己运行：

```bash
go run slice_demo.go
```

建议你重点观察两个现象：

1. `other[1] = 99` 后，`nums` 也变了
2. `append` 要把返回值重新赋给 `nums`

## 练习题

目标：

练习切片创建、共享底层数据，以及 `append` 的基本用法。

要求：

1. 定义一个 `[]string` 切片，打印内容和长度
2. 定义一个 `[]int` 切片并赋值给另一个变量
3. 修改副本中的元素，观察原切片变化
4. 使用 `append` 再追加一个元素

### 参考答案代码

```go
package main

import "fmt"

func main() {
	cities := []string{"Beijing", "Shanghai", "Shenzhen"}
	fmt.Println(cities)
	fmt.Println(cities[0])
	fmt.Println(len(cities))

	nums := []int{1, 2, 3}
	other := nums
	other[1] = 99
	fmt.Println("nums:", nums)
	fmt.Println("other:", other)

	nums = append(nums, 4)
	fmt.Println("after append:", nums)
}
```

### 预期运行结果

```text
[Beijing Shanghai Shenzhen]
Beijing
3
nums: [1 99 3]
other: [1 99 3]
after append: [1 99 3 4]
```

## 本节小结

这一节你先记住 4 句话：

1. **`[]T` 是切片，不是数组**
2. **切片通常描述的是底层数组的一段数据**
3. **切片比数组更适合处理动态长度的数据**
4. **`append` 会返回新的切片值，所以要接住返回值**

你现在学到的位置是：**已经从数组过渡到切片，开始接触 Go 最常用的集合类型**。

下一步最自然的内容是：**`make` 创建切片、`len/cap` 的直观理解，以及切片扩容时会发生什么**。
